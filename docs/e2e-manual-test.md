# 端到端手动测试手册

按顺序走完所有交互模式。每个步骤有预期行为，标记 ✅ 表示通过。

---

## 准备

### 0.1 启动 Nocturne Memory

```bash
cd nocturne_memory
python backend/main.py
```

预期：终端显示 `Uvicorn running on http://127.0.0.1:8233`

```bash
# 验证
curl http://127.0.0.1:8233/health
# → {"status":"ok"}
```

### 0.2 启动 pi 加载运行时

```bash
cd ..
pi -e .pi/extensions/runtime/index.ts
```

预期：进入 pi 交互界面，显示可用命令提示。

### 0.3 列出故事包

```
/stories-list
```

预期输出：
```
📚 已有 1 个故事包:
1. 卫宫士郎-日常沙盒  ○ 未开始
```

## 1. GM 模式 — 舞台模式（核心叙事流程）

### 1.1 进入故事

```
/stories-play 卫宫士郎-日常沙盒
```

预期：
- 显示 `📖 进入故事：卫宫士郎-日常沙盒`
- 显示世界观摘要（冬木市、海滨城市……）
- 显示 `🎭 角色已就绪：卫宫士郎, 远坂凛`
- 显示开场白（清晨六点的厨房场景）
- 问你"要开始吗？"

✅ 通过标记：________

### 1.2 new-turn — 广播场景给角色

使用 GM 工具 new-turn：

```
new-turn({
  content: "清晨六点，天色还没有完全亮透。\n厨房里传来菜刀落在砧板上的笃笃声，味噌汤的香气正从锅盖缝隙中袅袅升起。\n窗外院子里，冬青的叶子上凝着晨露。"
})
```

预期：
- 输出格式：`[卫宫士郎] act: ……` 和 `[远坂凛] act: ……`
- 士郎的响应使用第一人称（"我"、"我的"）
- 凛的响应使用第一人称
- 出现「」包裹的对白
- 内容与厨房/早晨场景相关

✅ 通过标记：________

### 1.3 observe — 查看角色状态

```
observe({ char: "卫宫士郎" })
```

预期：显示士郎最近一次响应内容

```
observe({ char: "远坂凛" })
```

预期：显示凛最近一次响应内容

✅ 通过标记：________

### 1.4 judge — 冲突仲裁（可选）

如果 new-turn 返回的角色意图明显冲突（如一个想离开、一个想挽留），调用：

```
judge({
  scene: "厨房里，士郎做好了早饭，凛刚走进来",
  intents: "[卫宫士郎] act: 「凛，过来坐吧，趁热吃。」\n[远坂凛] act: 「我先走了，今天学生会要开会。」"
})
```

预期：
- 输出仲裁结果
- 按角色性格判定（凛是否会真的不吃饭就走）
- 或输出"无冲突"

✅ 通过标记：________

### 1.5 write-story — 产出正史叙事

```
write-story({
  scene: "清晨六点半，卫宫家的厨房",
  intents: "[卫宫士郎] act: 「凛，今天来得真早啊。」我一边盛味噌汤一边回头看她。\n[远坂凛] act: 「哼，你不是每天都这个时间做早饭吗。」她在餐桌前坐下，翻开了一本看起来像魔术书的东西。",
  direction: "士郎和凛共进早餐，展现两人的日常互动节奏"
})
```

预期：
- 输出第三人称叙事文本
- 叙事优美流畅，无思考过程或元评论
- 正确将"我"转为角色名
- 不包含 `<think>` 标签
- 约 100-500 字

✅ 通过标记：________

### 1.6 archive-scene — 归档场景

```
archive-scene({
  sceneName: "清晨的厨房",
  summaries: JSON.stringify({
    "卫宫士郎": "早上我做了早饭，凛过来一起吃。她嘴上说不是来吃饭的，但还是坐下了。",
    "远坂凛": "我去卫宫家蹭了早饭。那个笨蛋的味噌汤确实做得不错。"
  }),
  narrative: "清晨六点半，卫宫家的厨房里弥漫着味噌汤的香气。士郎围着围裙站在灶台前，凛已经坐在餐桌旁，看似在看书其实在等开饭。"
})
```

预期：
- 显示 `📦 场景已归档: 20260710_清晨的厨房`
- 显示 `角色: 卫宫士郎, 远坂凛`
- 显示 `History: 写入 2 个角色`
- 显示 `Story-log: 已追加`

验证 story-log.md：
```bash
cat pi-characters/卫宫士郎-日常沙盒/story-log.md
# 应包含刚归档的叙事内容
```

验证记忆系统：
```bash
curl http://127.0.0.1:8233/api/browse/node?namespace=卫宫士郎\&domain=history\&path=scenes
# 应包含已归档的场景节点
```

✅ 通过标记：________

### 1.7 多轮互动

重复 1.2 → 1.5 → 1.6 至少三轮，验证：
- 故事在推进（时间/地点变化）
- write-story 的叙事能衔接上一轮
- 角色响应基于场景变化

✅ 通过标记：________

### 1.8 退出故事

```
/stories-stop
```

预期：显示"故事已退出"

✅ 通过标记：________

## 2. GM 模式 — 每日模式

### 2.1 再次进入故事

```
/stories-play 卫宫士郎-日常沙盒
```

### 2.2 daily-start — 启动每日模式

```
daily-start({ startHour: 7 })
```

预期：
- 显示 `🌅 每日模式已启动`
- 显示 `时间: 07:00 (早晨)`
- 显示 `角色: 卫宫士郎, 远坂凛`

✅ 通过标记：________

### 2.3 daily-tick — 推进时间

```
daily-tick({})
```

预期：
- 显示 `⏱ 08:00 (早晨) 第 1 轮`
- 显示士郎和凛的自主行动摘要
- 响应内容与时间/场景相关（早上在做什么）

连续 tick 多次：
```
daily-tick({})  → 09:00 (午前)  上学
daily-tick({})  → 10:00 (午前)
daily-tick({})  → 11:00 (午前)
daily-tick({})  → 12:00 (中午)  午饭
daily-tick({})  → 13:00 (午后)
daily-tick({})  → 14:00 (午后)
```

预期：
- 跨时间块时自动切换（早晨 → 午前 → 中午 → 午后）
- 每个角色根据时间做出合理行动
- 角色行为符合其性格和身份

✅ 通过标记：________

### 2.4 daily-status — 查看状态

```
daily-status({})
```

预期：
- 显示当前时间、块名、已推进轮数
- 显示活跃角色数

✅ 通过标记：________

### 2.5 daily-stop — 停止

```
daily-stop({})
```

预期：显示 `🌙 每日模式已停止（共推进 X 轮）`

✅ 通过标记：________

### 2.6 切回舞台模式

停止每日模式后，直接用舞台工具：

```
new-turn({ content: "午后的阳光斜照进客厅，茶几上摊着一本翻到一半的魔术书。" })
```

预期：角色正常响应，不受之前每日模式影响

✅ 通过标记：________

## 6. 破限注入验证

### 6.1 检查角色是否有 preamble 记忆

通过 `fixer` 查看士郎的 session 文件（需要找到临时目录）：

```bash
ls /tmp/char-卫宫士郎-*/session/
# 应包含 preamble.jsonl 文件
cat /tmp/char-卫宫士郎-*/session/preamble.jsonl | head -5
# 第一行是 session header
# 后续是 user/assistant 消息对
```

### 6.2 验证角色在对话中体现 preamble 关系

发一个触发 preamble 对话的 env：

```
new-turn({
  content: "凛推开卫宫家的门走了进来。\n「哟，还没做好吗？我的腌萝卜呢？」"
})
```

预期：
- 士郎的回应应该自然地接上 preamble 中建立的关系（凛经常来蹭饭，士郎习惯了她这样）
- 不会出现自我介绍或重新建立关系
- 呈现"已经认识很久"的感觉

如果角色说"你是谁？"或自我介绍 → ❌ 破限注入失败

✅ 通过标记：________
