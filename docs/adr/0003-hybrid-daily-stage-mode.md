# 日常/舞台混合模式，而非纯舞台架构

角色不是在场景启动时 spawn、场景结束时销毁的"舞台演员"。角色有两种运行模式：**日常模式**（独立生活，有自主 agenda，不阻塞等待）和**舞台模式**（高密度互动，共享环境叙事，启用打断机制）。GM 通过环境信号自然吸引角色进入舞台，而非强制切换。

## 背景

最初的架构偏向戏剧舞台模型——一个场景 = 一个地点 = 一群固定角色在同一房间互动。场景有明确的起止，角色在场景开始时 spawn，结束时销毁。但实际使用表明：

1. **角色在场景之间也有生活感**——他们记得发生过的事，会反思，会在日记里写下自己的想法
2. **场景边界是模糊的**——角色可以离开一个地点走向另一个，新角色可以自然加入
3. **日常互动和舞台互动需要不同的节奏**——QQ 闲聊不应该有 wait/打断机制，深度场景需要

## 决定

### 两种模式

| 维度 | 日常模式 | 舞台模式 |
|------|----------|----------|
| Agent 生命周期 | 长期运行 | 日常中自然进入/退出 |
| 行为驱动 | 自主生活 agenda | 环境输入 + 打断机制 |
| 等待方式 | 不阻塞（自主推进 agenda） | 阻塞（`wait` 等待对方） |
| GM 叙事密度 | 低密度背景感知 | 高密度环境旁白 |
| 打断机制 | 不启用 | 启用（act 前检查新消息） |
| turn 边界 | 无明确 turn | 有明确的 turn 边界 |

### 进入舞台的方式（舞台邀请）

GM 不直接切换模式，而是：
1. 了解当前角色的 agenda 和位置
2. 创造环境信号——让某个 NPC 路过、让另一个角色出现在附近、安排一个需要互动的场景
3. 角色决定是否回应——如果回应，自然进入舞台模式；如果不回应，场景不成立，但不视为失败

### GM 注意力焦点

GM 一次只能聚焦在一个角色或一个地点上，给予高密度叙事。这一约束来自叙事注意力的自然限制——作者/GM 的注意力是有限的。未被聚焦的区域处于低密度的背景感知状态。

## 取舍

**取的**：角色有连续的生活感，而不是只在场景中"活着"。日常互动（QQ、chat）和深度互动（舞台）在同一个架构下统一。角色可以自主决定是否进入互动——尊重认知独立性。

**舍的**：架构更复杂（两种动作循环、模式切换逻辑、agenda 系统）。GM 需要管理注意力焦点的切换。舞台场景不再有精确可控的"开幕/闭幕"——角色可能在不合适的时候拒绝进入舞台。

**Status**: accepted
