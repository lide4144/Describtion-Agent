# NPC 由角色记忆驱动，而非独立记忆 namespace

NPC 没有自己的记忆 namespace。GM 扮演 NPC 时，通过主要角色对它的记忆来驱动行为。同一 NPC 在不同角色的记忆中有不同面貌时，GM 利用这种矛盾来深化 NPC 塑造，而非视为冲突。

## 背景

传统做法通常给每个角色（包括 NPC）独立的记忆存储，以保证行为一致性。但在互动故事中，NPC 只在与主要角色相遇时才"存在"——让 NPC 拥有完整生命周期和独立记忆既增加了系统复杂度，也浪费了认知隔离本身带来的叙事潜力：不同角色对同一个人的不同印象是最自然的人际冲突来源。

## 决定

- NPC 不分配独立的 memory namespace
- GM 扮演 NPC 时，查询当前场景中主要角色对该 NPC 的 core 记忆作为行为参考
- 当多个角色对同一 NPC 的记忆矛盾时，GM 主动利用这个矛盾为 NPC 赋予多层次的行为——它面对 A 时热情，面对 B 时冷淡，这与真人完全一致
- `history` 域中的场景归档可作为 GM 校正记忆偏差的参考依据

## 取舍

**取的**：叙事密度更高。矛盾记忆成为故事材料而非系统缺陷。架构更简洁（无需管理 NPC 记忆生命周期）。

**舍的**：NPC 缺少独立于角色的客观存在感。GM 需要额外工作来解析和合矛盾记忆。在极端情况下（一个角色彻底忘记了某个 NPC），GM 需要通过其他线索补全 NPC 行为。
